# 诊断方法论

> **用途**：Phase 3 定位问题章节时对照使用。

---

## 诊断流程

```
数据导入 → 指标计算 → 异常检测 → 问题定位 → 原因分析
```

---

## 异常检测方法

### 1. 追读率骤降检测

**定义**：某章追读率比前章下降超过 15%。

**检测方法**：
```
如果 第N章追读率 < 第(N-1)章追读率 × 0.85
则标记为异常
```

**可能原因**：
- 章节质量差（节奏拖沓、无冲突）
- 剧情走向不符预期
- 钩子失效

---

### 2. 完读率偏低检测

**定义**：某章完读率低于 60%。

**检测方法**：
```
如果 第N章完读率 < 60%
则标记为异常
```

**可能原因**：
- 章节太长（>5000字）
- 内容无聊（大段描写无剧情）
- 标题/简介误导

---

### 3. 互动率偏低检测

**定义**：某章互动率低于均值的 50%。

**检测方法**：
```
如果 第N章互动率 < 平均互动率 × 0.5
则标记为异常
```

**可能原因**：
- 章节平淡无奇，无讨论点
- 缺少争议/悬念
- 角色行为不讨喜

---

### 4. 趋势下滑检测

**定义**：连续 3+ 章关键指标持续下降。

**检测方法**：
```
如果 第N章 < 第(N-1)章 < 第(N-2)章
则标记为趋势下滑
```

**可能原因**：
- 剧情进入低谷期
- 长期问题积累
- 读者新鲜感消退

---

## 问题优先级

| 优先级 | 问题类型 | 严重程度 | 处理建议 |
|--------|---------|---------|---------|
| P0 | 追读率骤降 > 25% | 严重 | 立即修改章节 |
| P1 | 完读率 < 50% | 严重 | 立即修改章节 |
| P2 | 趋势连续下滑 | 中等 | 分析原因，调整后续 |
| P3 | 互动率偏低 | 低 | 可考虑优化 |

## 诊断报告结构

```yaml
diagnosis:
  total_chapters: 50
  analysis_date: 2026-06-29
  platform: 起点

  overall_metrics:
    avg_retention: 65%
    avg_completion: 72%
    avg_engagement: 3.5%

  problem_chapters:
    - chapter: 12
      issue: 追读率骤降
      severity: P0
      drop: 25%
      possible_cause: 节奏拖沓
      suggestion: 精简章节，增加冲突

    - chapter: 23
      issue: 完读率偏低
      severity: P1
      rate: 48%
      possible_cause: 章节太长
      suggestion: 拆分为两章
```

## 质量检查

| # | 检查项 | 通过标准 |
|---|--------|----------|
| 1 | 异常全覆盖 | 所有异常章节都已标记 |
| 2 | 优先级明确 | 每个问题有优先级 |
| 3 | 原因合理 | 原因分析符合逻辑 |
| 4 | 建议可执行 | 建议具体可操作 |
