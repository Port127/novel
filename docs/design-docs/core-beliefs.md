# Core Beliefs

本文件记录本仓库的共享 harness 信念。

## 来自 Harness Engineering
- 仓库是真实来源。
- `AGENTS.md` 是地图，而非百科全书。
- 通过链接文档实现渐进披露。
- 在可能处使用机械检查强制流程。

## 来自 Eval-driven Development
- 对关键行为尽早定义 eval。
- 优先使用确定性检查。
- 在改变提示词/流程前先读 transcript 与失败轨迹。
- 用显式、可复现信号跟踪质量。

## 本地适配
- 写作质量与合规流程既需要确定性信号，也需要编辑判断。
- 无可复现来源的指标必须标记 `TBD`。
