# AI 辅助网文创作 Copilot 体系架构设计 (Specs)

## 1. 核心理念与战略纠偏

### 1.1 摒弃"全自动流水线"与"工匠思维"陷阱
早期的设计试图打造一条无需人工干预的自动化流水线，但受限于当前 LLM 的长文本遗忘和多任务能力退化，强行自动化的产物必然是"充满 AI 味、缺乏网文爽感"的工业垃圾。
即便降维到"Copilot 模式"，仅仅提升"写"的环节也是不够的。本项目的最高宗旨是**"跑通商业化，真实写出能赚钱的优秀小说"**。网文的商业化是一个闭环，只有覆盖完整链条，才能称得上是赚钱的 Copilot。

### 1.2 确立"商业化闭环副驾 (Commercial Copilot)"模式
系统定位从单纯的"辅助写作"升级为**"赚钱流水线总装车间"**，覆盖从选题到反馈的全生命周期：
- **人类创作者（你）**：是绝对的主脑、产品经理和总导演，负责商业决策、审美判断和节奏把控。
- **Agent 体系**：是不互相自动对话、极度偏科的"专业编辑部员工"，负责从选题、开篇、日更到数据分析的各个专精节点。
- **`novel-material` 项目**：是编辑部的"弹药库与档案馆"，提供爆款参考、事实记忆、商业数据洞察以及**套路模板库**。

商业化全链路覆盖（按变现影响力排序）：
> **选题** → **开篇（黄金三章）** → **付费卡点设计** → **日更产能** → **数据闭环**

### 1.3 三条贯穿性设计原则

以下原则贯穿所有 Skill 的设计与实施，不是独立模块，而是底层约束：

#### 原则一：成本可控 (Cost Control)
每一次 LLM 调用都有成本。系统必须内置 token 预算控制器：
- **单 Skill 调用上限**：每个 Agent 单次调用设有 token 硬上限（按模型等级差异化配置）。
- **单章重试上限**：Writer ↔ Checker 之间的打回修改最多 2 轮，超出后强制人工介入。
- **每日成本预警**：当日累计 LLM 开销达到预警线（如 ¥15/天）时，自动降级所有非 P0 任务到便宜模型。

#### 原则二：品类驱动 (Genre-Driven)
不同品类的爽感机制完全不同（男频都市 vs 玄幻 vs 女频古言 vs 系统文）。所有创作类 Skill 必须：
- 启动前指定或继承 `genre_profile`（品类画像），加载对应的爽感维度权重与套路模板。
- 未指定品类时，系统应强制要求选择，不得以"通用模式"运行。

#### 原则三：存稿安全 (Manuscript Safety)
存稿是网文作者的命根子。系统必须：
- 对存稿进行分级管理（精修稿 / 粗稿 / 大纲稿）。
- 低于安全线时自动切换应急模式，暂停非紧急任务，集中算力赶存稿。
- 连续产出低质量内容时，主动建议请假——发一章水稿的傷害大於停更一天。

## 3. 核心 Agent 技能定义 (Skills)

将 Agent 技能按照**对收入的影响优先级 (P0~P2)** 进行全面重构。

### 3.1 【P0 级：生存与变现红线】

#### 3.1.1 `audit-hooks` (章末钩子 + 章节标题审查员 - 追读率保障) **[新增+增强]**
- **触发场景**：每一章写完、准备发布前。
- **核心逻辑**：
  - **钩子检测**：追读率是平台的命根子。扫描本章最后 500-800 字（而非仅末尾 200 字），检测是否以悬念、反转或危机结尾。同时检测全文爽点密度（例如每 1000 字是否有情绪拉升点），若连续 2000 字无冲突则发出"温吞水"警告，强制要求修改。
  - **章节标题检测**：章节标题直接影响读者点击率。检测标题是否包含悬念感、冲突感或信息差（如"他竟是……"比"第三章"点击率高数倍）。提供 3 个备选标题建议。
  - **品类适配**：不同品类的钩子偏好不同（玄幻偏好"突破悬念"，都市偏好"打脸预告"），检测标准需结合 `genre_profile`。

#### 3.1.2 `anti-ai-polish` (多层级去 AI 味审校 - 金牌编辑) **[升级核心护城河]**
- **触发场景**：扩写员生成段落后，或作者自写部分感觉水分大时。
- **核心逻辑**：从单一禁词表升级为**五层检测模型**：
  1. **词汇层**：封杀"不禁、宛如、映射、然而"等典型 AI 高频词。
  2. **句式层**：打破连续排比、四字成语堆砌，强制短句为主。
  3. **段落层**：消除总分总结构和结尾强行升华。
  4. **叙事层**：修正上帝视角随意切换，补充口语化对话。
  5. **情感层**：注入网文特有的"尖锐情绪"（突然的暴怒、极致的憋屈）。

#### 3.1.3 `daily-update-workflow` (日更效率管家 - 速度即金钱) **[新增+增强]**
- **触发场景**：每日开工时。
- **核心逻辑**：编排与追踪写作流程，保证日更 6000-8000 字的产能。
  - **双模式运行**：提供"快速模式"（只调扩写+去AI味，<30分钟/章）和"精细模式"（全链路，<60分钟/章）。
  - **成本硬约束（新增）**：
    - 内置 token 预算控制器。每个 skill 单次调用设有 token 上限（按模型等级差异化配置，如便宜模型 ≤ 4K tokens，贵模型 ≤ 8K tokens）。
    - Writer ↔ Checker 之间最大重试 2 轮，超出后强制人工介入，不做无限循环。
    - 每日累计 LLM 开销达到预警线（默认 ¥15/天，可配置）时，自动降级所有非 P0 任务到便宜模型。
    - 输出每章成本报告（各 skill 调用的 token 数和费用明细）。
  - **存稿分级管理（新增）**：
    - 存稿分为三级：**精修稿**（已通过 anti-ai-polish，可直接发布）、**粗稿**（已扩写但未精修，需过审校）、**大纲稿**（仅有节拍表，需完整扩写流程）。
    - 每级存稿独立计数，每日发布时优先消耗精修稿。
  - **应急模式（新增）**：
    - 黄色预警（存稿 ≤ 9 章）：暂停新书筹备、套路研究等非紧急任务，所有算力集中赶存稿。
    - 红色预警（存稿 ≤ 5 章）：自动切换为"快速模式"为默认，暂停精细模式，必要时建议请假。
    - **请假建议**：当检测到连续 3 章质量评分低于阈值，或当日单章耗时异常（过长可能卡文、过短可能水稿），主动建议"今日请假，明日补更"。

#### 3.1.4 `design-paywall` (付费卡点设计师 - 收入命门) **[新增]**
- **触发场景**：大纲完成后、正式进入日更前。新书必用，老书付费数据回检时复用。
- **核心逻辑**：
  - **卡点位置计算**：分析大纲，找到最优付费切割点。理想卡点应满足：
    - 前一个爽点刚刚兑现（读者情绪处于高位）。
    - 一个新的大悬念刚刚抛出（读者好奇心被吊起）。
    - 主角的"核心卖点"已经充分展示（读者已建立期待）。
    - 绝对避开：主角低谷期、日常过渡期、新设定铺垫期。
  - **过渡章节拍设计**：付费前最后一章（免费末章）和付费后第一章（付费首章）是转化率的关键。
    - **免费末章**：必须做到"爽点兑现 + 致命悬念"双保险。读者合上这一章时，必须"不花钱就难受"。
    - **付费首章**：开头 200 字内必须立刻给出爽感反馈（读者刚花了钱，不能让他觉得亏了）。随后展开新弧线，让读者觉得"这钱花得值"。
  - **数据回检**：结合 `novel-material` 的 `DataDB`，分析同类书籍的付费转化率数据，验证卡点策略。
  - **品类适配**：不同品类的"最佳卡点套路"不同（玄幻偏好在突破前夕切付费，都市偏好在打脸前夕切付费），需结合 `genre_profile`。

### 3.2 【P1 级：做对多赚钱】

#### 3.2.1 `forge-golden-chapters` (黄金三章锻造师 - 决定流量池) **[新增]**
- **触发场景**：新书开篇前三章（约 6000-9000 字）。
- **核心逻辑**：前三章决定生死。强制执行高精度节奏设计：300 字内出第一冲突，前三章完成"人设立住+金手指亮相+第一个小高潮"。不跑常规扩写，而是按极密集的微节拍（Micro-beats）逐段生成。
- **品类适配**：不同品类的黄金三章侧重点不同：
  - 玄幻：开局废柴受辱 → 金手指觉醒 → 首次反击。
  - 都市：开局被退婚/背叛 → 隐藏身份曝光 → 第一波打脸。
  - 系统文：系统激活 → 首个任务 → 奖励碾压。

#### 3.2.2 `scout-topic` (选题侦察兵 - 寻找蓝海) **[新增+增强]**
- **触发场景**：开新书、找题材时。
- **核心逻辑**：
  - **榜单分析**：调用 `novel-material` 分析平台畅销榜、飙升榜，识别"热门但竞争不饱和"的标签组合，反推爆款上架时的题材窗口期，从源头提高天花板。
  - **手动数据导入（新增）**：支持通过 `nm import ranking --file <path>` 手动导入榜单数据（CSV/JSON 格式）。不强制依赖爬虫，降低初期实施阻力。导入后自动解析标签、排名、字数等字段。
  - **竞品分析**：对候选题材，输出竞品数量、头部作品数据、市场饱和度评估。
  - **品类适配**：分析结果按品类隔离，避免跨品类数据污染。

#### 3.2.3 `design-character` (爽感人设主编) **[增强]**
- **触发场景**：设定新主角、反派或重要配角时。
- **核心逻辑**：在好莱坞编剧理论（执念、致命弱点）基础上，**新增网文爽感维度**：
  - 装逼打脸指数（是否适合扮猪吃虎）
  - CP 感/嗑点
  - 反派恶心度
  - 金手指机制与限制
  - **品类适配（新增）**：接受 `genre_profile` 参数（格式定义见 Section 4.4），不同品类加载不同的爽感维度权重。例如：
    - 男频都市：打脸指数权重最高，CP 感次之。
    - 女频古言：CP 感权重最高，角色关系网络次之。
    - 玄幻：金手指机制权重最高，升级速度感次之。
  - **套路模板关联（新增）**：人设设计完成后，自动匹配适用的叙事套路模板（如"扮猪吃虎型主角"自动关联 `trope/pretend-weak` 模板）。

### 3.3 【P2 级：质量与长线保障】

#### 3.3.1 `ask-architect` (剧情与节奏架构师) **[增强]**
- **触发场景**：卡文、构思新大纲、规划新副本。
- **核心逻辑**：
  - **量化张力曲线**：确保每 5-8 章小高潮，20-30 章大高潮，高潮前有 2-3 章压抑。连续多章"日常"将触发节奏掉线警告。
  - **套路模板调用（新增）**：不再从零构思，而是先查 `novel-material` 的套路模板库（`nm search trope`），找到适用的经典叙事模板，在其基础上微调。模板提供标准节拍表（Beat Sheet），大幅降低构思成本。
  - **品类适配（新增）**：节奏参数按品类调整。玄幻的升级节奏（每 10-15 章一个小境界突破）和都市的打脸节奏（每 3-5 章一次反转）完全不同。

#### 3.3.2 `analyze-stats` (数据诊断师 - 数据闭环) **[新增]**
- **触发场景**：每发完 5-10 章后，或发现数据异动时。
- **核心逻辑**：输入平台后台数据（追读率、跳出率、**付费转化率**）。定位掉追读的具体章节，结合上下文诊断原因（节奏拖沓？主角怂了？反派降智？），并联动 `ask-architect` 调整后续大纲。
- **付费转化分析（新增）**：重点分析免费→付费的转化率，若低于品类均值，触发 `design-paywall` 复盘卡点设计。

#### 3.3.3 `flesh-out-chapter` (节拍扩写员) **[优化]**
- **触发场景**：有章节走向，需扩写段落时。
- **核心逻辑**：废除模糊的"极小范围"，引入**节拍表 (Beat Sheet) 机制**。用户定义本章节拍，AI 严格在当前节拍内扩写，杜绝越界推动未授权剧情。

#### 3.3.4 `check-logic` (事实核查员)
- **触发场景**：写完重要章节后，防吃书。
- **核心逻辑**：利用 `novel-material` 的 `nm search event/detail` 比短事实库，指出硬逻辑 Bug。

## 4. `novel-material` 的深度集成机制

`novel-material` 作为核心弹药库，需支持更丰富的检索命令以支撑商业化需求：

### 4.1 向外求索（外部经典案例检索）
- `nm search insight`：提取手法、套路。
- `nm search opening`：分析爆款开局（支撑 `forge-golden-chapters`）。
- `nm search pacing`：分析追读率拐点（支撑 `ask-architect`）。
- `nm search character`：查询爆款反派与主角互动模式（支撑 `design-character`）。
- `nm search trope`：**[新增]** 检索套路模板库，按情绪机制（如"压抑→爆发→震惊"）或叙事类型（如"扮猪吃虎"、"退婚流"）匹配经典模板。

### 4.2 向内审视（当前作品动态查询）
- `nm storage sync`：增量同步已完稿章节。
- `nm search chapter`：获取精准历史摘要，防遗忘。
- `nm search detail`：验证设定（支撑 `check-logic`）。

### 4.3 套路模板库 **[新增]**

独立于爆款数据库，专门存储**可复用的叙事模板**。每个模板包含：
- **模板 ID**：如 `trope/pretend-weak`（扮猪吃虎）。
- **情绪机制**：标准情绪曲线描述（如：被轻视→隐忍→爆发→围观震惊→装逼收场）。
- **标准节拍表**：拆分为 5-8 个微节拍，每个节拍有明确的情绪目标和字数建议。
- **适用品类标签**：该模板最适合的品类（如玄幻、都市）。
- **经典案例引用**：关联到爆款数据库中的具体作品章节。

已有模板（首批）：
| 模板 ID | 名称 | 情绪机制 | 适用品类 |
|---------|------|---------|---------|
| `trope/pretend-weak` | 扮猪吃虎 | 被轻视→隐忍→实力碾压→围观震惊 | 玄幻、都市 |
| `trope/broken-engagement` | 退婚流 | 被羞辱→发誓→逆袭→打脸 | 玄幻、都市 |
| `trope/auction-house` | 拍卖会 | 层层竞价→一锤定音→后续麻烦 | 玄幻、仙侠 |
| `trope/sect-tournament` | 宗门大比 | 不被看好→一路碾压→决赛逆转 | 玄幻、仙侠 |
| `trope/identity-reveal` | 身世揭秘 | 伏笔→真相大白→众人震惊 | 全品类 |
| `trope/face-slap-escalation` | 递进打脸 | 小反派→中反派→大反派，层层递进 | 全品类 |

### 4.4 品类画像配置 **[新增]**

每个品类维护一份画像文件，包含：
- **品类 ID**：如 `genre/urban`（都市）、`genre/xuanhuan`（玄幻）、`genre/system`（系统文）。
- **爽感维度权重**：各爽感维度的权重分配（打脸、升级、CP、悬念、智商博弈等）。
- **节奏参数**：小高潮间隔（章数）、大高潮间隔、付费卡点偏好位置。
- **黄金三章模板**：该品类开篇三章的标准套路。
- **禁忌清单**：该品类读者最不能容忍的毒点（如玄幻的"主角长期受虐"、都市的"后宫修罗场"）。

### 4.5 手动数据导入 **[新增]**

- `nm import ranking --file <path> --platform <platform>`：导入平台榜单数据（CSV/JSON 格式）。
- `nm import stats --file <path>`：导入作者本人的后台数据（追读率、付费转化率等）。
- 导入后自动解析、校验、入库，支持增量更新和去重。

## 5. 商业化质量门禁 (Commercial Quality Gates)

以下门禁是贯穿整个系统的底层约束，独立于具体 Skill，在运行时由 `daily-update-workflow` 或 `BaseCommercialSkill` 统一执行。

### 5.1 Token 预算控制

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `token_limit_per_call.cheap` | 4,000 | 便宜模型（如 Haiku）单次调用 token 上限 |
| `token_limit_per_call.mid` | 6,000 | 中级模型（如 Sonnet）单次调用 token 上限 |
| `token_limit_per_call.expensive` | 8,000 | 贵模型（如 Opus）单次调用 token 上限 |
| `max_retry_per_chapter` | 2 | Writer ↔ Checker 之间最大打回重试次数 |
| `daily_cost_alert` | ¥15 | 每日累计 LLM 开销预警线（可配置） |
| `cost_report_per_chapter` | true | 是否在每章完成后输出成本明细报告 |

当触发预警线时，系统行为：
1. 暂停所有 P2 级任务。
2. P1 级任务降级到便宜模型。
3. P0 级任务保持当前模型，但输出警告提示。

### 5.2 品类路由门禁

任何创作类 Skill（`forge-golden-chapters`、`ask-architect`、`design-character`、`design-paywall`、`audit-hooks`）启动前，必须满足：
1. 检查当前项目是否已配置 `genre_profile`。
2. 若未配置，**强制阻断**并提示用户选择品类。
3. 已配置后，将品类参数注入 Skill 的上下文，影响爽感权重、节奏参数、套路模板选择。

### 5.3 存稿应急协议

| 存稿水位 | 预警等级 | 系统行为 |
|----------|---------|---------|
| ≥ 10 章 | 🟢 正常 | 所有模式正常运行 |
| 6-9 章 | 🟡 黄色预警 | 暂停新书筹备、套路研究等非紧急任务；所有算力集中赶存稿 |
| ≤ 5 章 | 🔴 红色预警 | 默认切换为"快速模式"；暂停精细模式；若连续 3 章质量评分低于阈值，建议请假 |
| 0 章 | ⚫ 断更 | 全面停工，仅保留事实核查和去AI味能力，全力抢救存稿 |

**请假建议触发条件**（满足任一即触发）：
- 连续 3 章的 `anti-ai-polish` 五层综合评分低于 60 分（五层各 20 分，满分 100 分；评分由 `anti-ai-polish` 在审校时自动输出）。
- 当日单章耗时异常（> 90 分钟可能卡文，< 15 分钟可能水稿）。
- 作者主动标记"今日状态不好"。
