```markdown
# 黄金三章核心规则

> **用途**：Phase 2-4 锻造前三章时对照使用。
> **核心目标**：让读者在前三章内完成"留下来"的决定。

---

## 三章功能定义

| 章节 | 核心功能 | 必须完成 | 禁止事项 |
|------|---------|---------|---------|
| 第一章 | 钩住读者 | 300字内出冲突/钩子；立住主角人设；让读者知道"讲什么" | 大段设定交代、回忆、旁白解说 |
| 第二章 | 展示核心机制 | 金手指/核心设定完整亮相；展示机制限制；建立期待 | 新角色大量涌入、偏离主线 |
| 第三章 | 第一个爽点 | 首个小高潮；让读者看到"爽"的可能；建立长线期待 | 高潮后立刻接平淡、主角被动挨打 |

---

## 第一章规则

### 规则 1：300 字内出冲突

**定义**：读者在前 300 字内必须遇到"不对劲"的事——冲突、悬念、异常。

**为什么**：番茄读者平均 3 秒决定是否继续。300 字 = 阅读约 30 秒。如果 300 字内没有钩子，读者划走。

**冲突类型**：

| 类型 | 示例 | 适用品类 |
|------|------|---------|
| 人际冲突 | 被嘲讽、被退婚、被陷害 | 玄幻、都市 |
| 生存冲突 | 濒死、被困、被追杀 | 玄幻、悬疑 |
| 身份冲突 | 重生发现时间点、穿越到奇怪的身体里 | 重生、穿越 |
| 异常事件 | 系统激活、收到神秘短信、看到不该看的 | 系统、悬疑 |
| 情感冲突 | 尴尬相遇、误会、撞见秘密 | 言情 |

**错误示例**：
> 天玄大陆，万族林立。修炼之道，以灵气为基。灵气分为天地玄黄四个等级……

→ 纯设定铺陈，无冲突，读者跑光。

**正确示例**：
> 林浩被一脚踹飞出去，撞在演武场的石柱上，嘴里涌出一口鲜血。
> "废物也配修炼？"

→ 第一句就有冲突，读者想知道"为什么""后来怎样"。

### 规则 2：主角人设立住

**定义**：读者在第一章结束时，能用一句话说出"主角是个什么样的人"。

**方法**：
- 用一个具体行为展示性格（不是形容词堆砌）
- 面对冲突时的选择定义角色

**示例**：
- 被侮辱后不还嘴，但默默记下 → 隐忍型
- 明知打不过还要上 → 热血型
- 被陷害后冷静分析 → 智谋型

### 规则 3：让读者知道"讲什么"

**定义**：第一章结束后，读者能判断"这本书是讲 XX 的，我要不要继续看"。

**方法**：
- 核心设定/金手指在第一章暗示或亮相
- 主线方向明确（复仇/升级/恋爱/破案）

---

## 第二章规则

### 规则 4：核心机制完整亮相

**定义**：金手指/系统/重生优势在第二章必须完整展示其功能和运作方式。

**展示要素**：
1. **功能**：金手指能做什么？
2. **限制**：金手指不能做什么？/代价是什么？
3. **使用方法**：主角怎么触发/使用？

**为什么**：读者需要知道"主角靠什么混"，才能产生期待。

### 规则 5：建立期待

**定义**：第二章结束时，读者对"主角用这个能力能做到什么"产生好奇。

**方法**：
- 展示金手指的潜力（但不完全释放）
- 设置一个"如果用得好，能解决 XX 问题"的目标
- 暗示更大的可能性

---

## 第三章规则

### 规则 6：第一个小高潮

**定义**：主角在第三章首次展示实力/魅力，打败第一个障碍。

**高潮设计**：
- 对手不能太弱（否则打赢了也没爽感）
- 过程要有波折（不能一招秒）
- 结果要让读者觉得"主角有前途"

**小高潮模式**：

| 模式 | 结构 | 示例 |
|------|------|------|
| 打脸模式 | 被小看→展示实力→对方震惊 | 演武场反杀 |
| 智取模式 | 陷入困境→找到破绽→逆转 | 利用规则反杀 |
| 意外模式 | 意外事件→主角应对→获得认可 | 突发危机中救人 |

### 规则 7：建立长线期待

**定义**：第三章结束时，读者对后续剧情产生持续期待。

**方法**：
- 展示更大的世界/更强的对手
- 设置中期目标（进入内门/赢得比赛/解开谜团）
- 埋设长线伏笔
```

```markdown
## 什么是微节拍

**定义**：一个微节拍 = 一个自然段的核心功能。

**与节拍的关系**：
- 节拍（Beat）：章节级，一个场景/事件 = 一个节拍
- 微节拍（Micro-Beat）：段落级，一个自然段 = 一个微节拍

**示例**：
```
节拍：动作 - 主角冲进大厅
├─ 微节拍1：推门动作（1句）
├─ 微节拍2：大厅场景扫描（2句）
├─ 微节拍3：发现敌人位置（1句）
└─ 微节拍4：迈步向前（1句）
```

## 段落节奏规则

### 规则 1：一段一事

**核心**：每个自然段只承载一个动作、一个信息变化或一组紧密相关的反应。

```
错误（一段多事）：
他推开门，看到房间里坐满了人，所有人的目光都投向他。
他走到中间的座位坐下，发现桌上放着一封信。他拆开信，
上面写着一行字——"你被选中了。"他抬起头，发现所有人
都在笑。

正确（一段一事）：
他推开门。

房间里坐满了人，所有目光投向他。

他走到中间的座位坐下。桌上放着一封信。

"你被选中了。"

他抬起头。所有人都在笑。
```

### 规则 2：长短交替

**核心**：连续段落不能都是同一长度，要长短交替形成节奏。

```
节奏模式：
短（1句）→ 中（2-3句）→ 短（1句）→ 长（3-5句）→ 短（1句）
```

**不同场景的节奏**：

| 场景 | 节奏模式 | 示例 |
|------|---------|------|
| 打斗/紧张 | 短-短-短 | 他出拳。对方闪开。又是一拳。 |
| 对话 | 中-短-中 | 对话+动作描写交替 |
| 氛围营造 | 中-长-短 | 环境→感受→反应 |
| 推理分析 | 长-中-短 | 思考→线索→结论 |

### 规则 3：句长匹配情绪

**核心**：句子长度与情绪强度成正比。

| 情绪 | 句长 | 示例 |
|------|------|------|
| 紧张/恐惧 | 3-8 字 | 他跑。风灌进肺里。 |
| 愤怒/爆发 | 5-10 字 | 他一拳砸在桌上。碗跳起来。 |
| 平静/日常 | 8-15 字 | 厨房里传来炒菜的声音。 |
| 伤感/沉思 | 10-20 字 | 那些年发生的事，像放电影一样闪过。 |

### 规则 4：对话节拍规则

**核心**：对话不是独立的段落，必须搭配动作/表情/环境。

```
错误（纯对话）：
"你来了。"
"嗯。"
"坐吧。"
"谢谢。"

正确（对话+动作）：
"你来了。"她没抬头，继续翻手里的文件。

"嗯。"他站在门口，没进去。

"坐吧。"她把文件合上，推了推对面的椅子。

"谢谢。"他坐下，手指无意识地摩挲着杯沿。
```

## 微节拍编排模式

### 模式 1：动作序列

**适用**：打斗、追逐、紧张场景
**结构**：动作→感知→反应→动作→感知→反应

```
他一拳砸过去。（动作）
拳风带着血腥味。（感知）
对方纹丝不动。（反应）
```

### 模式 2：对话推进

**适用**：日常、谈判、冲突对话
**结构**：对话+动作→对话+反应→信息揭示→对话+情绪

```
"你知道错在哪吗？"他把报告扔在桌上。（对话+动作）

他看着散落的纸张，没说话。（反应）

"第三页的数据是假的。"（信息）

"我知道。"他终于开口，声音很平。（对话+情绪）
```

### 模式 3：氛围沉浸

**适用**：场景建立、情绪铺垫
**结构**：环境→感知→情绪→动作

```
巷子里很暗。路灯坏了一半。（环境）

空气里有股潮湿的味道，混着下水道的臭气。（感知）

他皱了皱鼻子。（情绪/反应）

加快脚步往里走。（动作）
```

### 模式 4：信息渗透

**适用**：交代背景/设定信息
**结构**：动作中嵌入信息→角色反应→动作继续

```
他掏出手机看了一眼。三条未读消息，都是公司群发的。
（动作+信息渗透：暗示公司氛围）

他划掉消息，打开了另一个 APP。（反应：暗示不关心公司）
```

## 段落密度控制

### 每段字数参考

| 场景类型 | 段落均长 | 说明 |
|---------|---------|------|
| 打斗/紧张 | 5-15 字/段 | 快节奏，多分段 |
| 对话 | 10-30 字/段 | 对话+动作标签 |
| 日常叙事 | 15-40 字/段 | 中等节奏 |
| 环境描写 | 20-50 字/段 | 慢节奏，可稍长 |
| 心理活动 | 10-30 字/段 | 避免大段心理 |

### 段落长度禁区

| 类型 | 字数 | 处理 |
|------|------|------|
| 过短 | < 3 字（非对话） | 除非特殊节奏需要，否则合并 |
| 过长 | > 80 字 | 按动作/信息变化拆分 |
| 全篇同长 | 每段都是 20-30 字 | 人为制造长短差异 |

| 纯对话段 | 连续多段纯对话 | 插入动作/表情标签 |
| 全篇同长 | 每段字数一致 | 人为制造节奏差异 |
| 心理独白过长 | 连续 > 2 段心理描写 | 用动作/对话打断 |
```

```markdown
## 核心规则

### 规则 1：段落密度

- 一段通常只承载一个动作、一个信息变化或一组紧密相关的反应。
- 逗号串太长、多个完整动作挤在一段里时，按动作或信息变化拆开。
- 连续短段碎成提纲时，合并同一镜头内的相邻句。

### 规则 2：动作 + 对话 + 情绪反应

三要素循环推进，不要单写心理活动超过 2 段。

情绪不用"他感到/他觉得"，用身体反应和行为表现：
- 不写"他很紧张" → 写"手心全是汗，筷子差点掉了"
- 不写"她很愤怒" → 写"她把杯子摔在地上，碎片弹到脚背上也没弯腰捡"
- 不写"他很伤心" → 写"他在车里坐了二十分钟才发动引擎"

### 规则 3：短句优先

| 场景 | 句长 | 示例 |
|------|------|------|
| 打斗/紧张 | 3-8 字 | 他侧身。刀光一闪。血溅在墙上。 |
| 对话 | 口语化 | "你疯了？""可能吧。" |
| 日常 | 8-15 字 | 厨房里有炒菜的声音，闻着像辣椒炒肉。 |

### 规则 4：口语化表达

- 对话不要书面语（"我认为此事不妥" → "我觉得不靠谱"）
- 叙述也不要端着（"他目光如炬" → "他眼珠子一动不动盯着"）
- 短语优先于成语（"无可奈何" → "没办法"）

## 8 种 AI 写作模式检测

### 模式 1：AI 高频词
见上方禁用词表。

### 模式 2：弱化副词泛滥
微微/淡淡/缓缓/轻轻，每 1000 字 ≤ 3 个。

### 模式 3：意义膨胀
"意义深远" → 写具体后果。"前所未有" → 给出对比参照。

### 模式 4：万能结论
"未来可期" → 用未解决的紧张感结尾。"前途无量" → 删。

### 模式 5：论文体段落结构
小说中出现"不难看出""由此可见""事实上""综上所述" = AI 入侵。

### 模式 6：书面语连词泛滥
"于是乎""与此同时""从而""因而""诚然" → 口语化或删除。

### 模式 7：三连排比癖
AI 喜欢凑成三个以显"完整" → 砍到只剩最有力的一条。

### 模式 8：解释腔/上帝视角/安排感
叙述者跳出角色去解释、剧透、总结、定性 → 删，改成角色视角内的呈现。

| 表现 | 处理 |
|------|------|
| 解释因果（"之所以…是因为"） | 删，让读者自己拼 |
| 上帝视角剧透（"她不知道的是"） | 删，只写角色此刻知道的 |
| 替读者下结论（"演得真好"） | 删，摆证据让读者判 |
| 安排感（为后文强行交代背景） | 用闪念/半句话带出 |

## 系统性去AI三遍法

### Pass 1：去泛化（Strip Generic）
- 抽象情绪总结句 → 删或替换为具体动作
- 假深度句 → 删
- 意义膨胀 → 缩小到具体影响
- 工整对比句式 → 打散重写
- 所有角色说话一样"高级" → 区分语气

**原则**：能删就删，不能删就用具体细节替换。这一遍去掉 80% 的 AI 味。

### Pass 2：去书面化（Cut Professional Diction）
- 分析性用词（"机制""结构""逻辑""体系"） → 换成日常表达
- 抽象名词滥用 → 直接说事
- 体制内用语（"进一步""深入""推进"） → 删

### Pass 3：回自然感（Restore Natural Presence）
- 具体的感官细节（气味、温度、触感）
- 角色说话方式的区分
- 节奏变化（长短句交错）
- 场景特有的记忆点

**原则**：少即是多。每段加 1-2 个具体细节就够了。
```

```javascript
#!/usr/bin/env node
// check-golden-structure.js — 黄金三章结构检查
// Usage: node check-golden-structure.js <scout_report.yaml> <chapter_001.md> [chapter_002.md] [chapter_003.md]

const fs = require('fs');
const yaml = require('js-yaml');

// 开篇钩子类型对应的首冲突字数阈值
const HOOK_THRESHOLDS = {
  golden_finger: 300,
  reborn_advantage: 300,
  conflict: 300,
  meet_cute: 500,
  mystery_hook: 500,
};

// 各章必需的关键结构元素
const CHAPTER_REQUIREMENTS = {
  1: {
    name: '第一章',
    required: ['首冲突/钩子', '主角出场', '主角人设展示'],
    description: '钩住读者',
  },
  2: {
    name: '第二章',
    required: ['核心机制展示', '机制限制展示', '期待建立'],
    description: '展示核心机制',
  },
  3: {
    name: '第三章',
    required: ['首个小高潮', '主角实力展示', '长线期待建立'],
    description: '第一个爽点',
  },
};

function main() {
  const scoutFile = process.argv[2];
  const chapterFiles = process.argv.slice(3);

  if (!scoutFile || chapterFiles.length === 0) {
    console.error('Usage: node check-golden-structure.js <scout_report.yaml> <chapter_001.md> [chapter_002.md] [chapter_003.md]');
    process.exit(2);
  }

  let scout;
  try {
    scout = yaml.load(fs.readFileSync(scoutFile, 'utf8'));
  } catch (err) {
    console.error(`无法读取 scout_report.yaml: ${err.message}`);
    process.exit(2);
  }

  const hookType = scout.required_elements?.opening_hook?.type || 'conflict';
  const genre = scout.genre || 'unknown';
  const hookThreshold = HOOK_THRESHOLDS[hookType] || 300;

  const findings = [];
  const chapters = [];

  // 读取各章内容
  for (let i = 0; i < chapterFiles.length; i++) {
    const file = chapterFiles[i];
    try {
      const content = fs.readFileSync(file, 'utf8');
      chapters.push({ number: i + 1, content, file });
    } catch (err) {
      findings.push({
        severity: 'blocking',
        chapter: i + 1,
        message: `无法读取第 ${i + 1} 章文件: ${file}`,
      });
    }
  }

  // 检查章节数量
  if (chapters.length < 3) {
    findings.push({
      severity: 'advisory',
      chapter: 0,
      message: `黄金三章需要 3 章，当前只有 ${chapters.length} 章`,
    });
  }

  // 逐章检查
  for (const ch of chapters) {
    const reqs = CHAPTER_REQUIREMENTS[ch.number];
    if (!reqs) continue;

    const text = ch.content;
    const charCount = countChineseChars(text);

    // 字数检查
    if (charCount < 1500) {
      findings.push({
        severity: 'blocking',
        chapter: ch.number,
        message: `第 ${ch.number} 章字数过少: ${charCount} 字（最少 1500 字）`,
      });
    }
    if (charCount > 6000) {
      findings.push({
        severity: 'advisory',
        chapter: ch.number,
        message: `第 ${ch.number} 章字数偏多: ${charCount} 字（建议不超过 5000 字）`,
      });
    }

    // 第一章专项检查：首冲突位置
    if (ch.number === 1) {
      const firstConflictPos = findFirstConflict(text, hookType);
      if (firstConflictPos > hookThreshold) {
        findings.push({
          severity: 'blocking',
          chapter: ch.number,
          message: `首冲突/钩子位置过晚: 第 ${firstConflictPos} 字（${hookType} 类型要求 ≤ ${hookThreshold} 字）`,
        });
      }

      // 检查是否有大段设定铺陈（连续 > 200 字纯描写无对话/动作）
      const longExposition = findLongExposition(text);
      if (longExposition) {
        findings.push({
          severity: 'advisory',
          chapter: ch.number,
          message: `开篇可能有设定铺陈：第 ${longExposition.pos} 字起连续 ${longExposition.len} 字无对话/动作`,
        });
      }

      // 检查主角是否出场
      if (!hasProtagonistIntro(text)) {
        findings.push({
          severity: 'advisory',
          chapter: ch.number,
          message: '主角出场不够明确，建议在首章建立主角存在感',
        });
      }
    }

    // 第三章检查：是否有小高潮（通过对话密度和动作密度估算）
    if (ch.number === 3) {
      const climaxDensity = estimateClimaxDensity(text);
      if (climaxDensity < 0.3) {
        findings.push({
          severity: 'advisory',
          chapter: ch.number,
          message: '第三章高潮感不足：动作/冲突密度偏低，建议增加紧张感',
        });
      }
    }

    // 通用检查：章末是否有钩子（最后 200 字是否有悬念/问句/转折词）
    const hasEndingHook = checkEndingHook(text);
    if (!hasEndingHook) {
      findings.push({
        severity: 'advisory',
        chapter: ch.number,
        message: '章末缺少钩子：最后 200 字无明显悬念/转折/期待',
      });
    }
  }

  // 输出结果
  if (findings.length === 0) {
    console.log(`✓ 黄金三章结构检查通过（品类: ${genre}，钩子: ${hookType}）`);
    process.exit(0);
  }

  for (const f of findings) {
    const prefix = f.chapter > 0 ? `第 ${f.chapter} 章: ` : '';
    console.log(`[${f.severity}] ${prefix}${f.message}`);
  }

  const hasBlocking = findings.some(f => f.severity === 'blocking');
  if (hasBlocking) {
    console.log(`\n✗ 发现 ${findings.filter(f => f.severity === 'blocking').length} 个阻断问题，需修复`);
    process.exit(1);
  } else {
    console.log(`\n△ 发现 ${findings.length} 个建议，可酌情调整`);
    process.exit(0);
  }
}

// 统计中文字符数
function countChineseChars(text) {
  const matched = text.match(/[一-鿿]/g);
  return matched ? matched.length : 0;
}

// 查找首冲突位置
function findFirstConflict(text, hookType) {
  // 冲突信号词
  const conflictSignals = [
    '踹', '打', '骂', '嘲', '笑', '怒', '吼', '砸', '推', '撞',
    '血', '痛', '死', '杀', '逃', '追', '喊', '叫', '惊', '吓',
    '系统', '激活', '觉醒', '重生', '穿越', '回来', '睁眼',
    '看到', '发现', '不对', '怎么回事', '什么', '谁',
    '相遇', '撞', '尴尬', '误会', '奇怪', '异常',
  ];

  // 按行扫描，找到第一个冲突信号
  const lines = text.split(/\n/);
  let charPos = 0;

  for (const line of lines) {
    for (const signal of conflictSignals) {
      if (line.includes(signal)) {
        return charPos;
      }
    }
    charPos += line.length;
  }

  return charPos; // 没找到，返回文末
}

// 查找长段设定铺陈
function findLongExposition(text) {
  const lines = text.split(/\n/);
  let noActionCount = 0;
  let startPos = 0;
  let currentPos = 0;

  for (const line of lines) {
    const trimmed = line.trim();

    // 对话行重置计数
    if (/[”"']/.test(trimmed) || /[:：]/.test(trimmed)) {
      noActionCount = 0;
      startPos = currentPos;
      currentPos += line.length;
      continue;
    }

    // 动作行重置计数
    if (/[打斗踢 punch kick hit]/.test(trimmed)) {
      noActionCount = 0;
      startPos = currentPos;
      currentPos += line.length;
      continue;
    }

    noActionCount += trimmed.length;

    if (noActionCount > 200) {
      return { pos: startPos, len: noActionCount };
    }

    currentPos += line.length;
  }

  return null;
}

// 检查主角是否出场
function hasProtagonistIntro(text) {
  // 检查是否有角色名或代词出现
  const hasProtagonist = /[他她]/.test(text.slice(0, 500));
  return hasProtagonist;
}

// 估算高潮密度（动作词+情绪词占比）
function estimateClimaxDensity(text) {
  const actionWords = ['打', '冲', '闪', '拳', '踢', '砸', '喊', '吼', '爆', '冲', '崩', '碎', '裂'];
  const emotionWords = ['怒', '恨', '惊', '震', '恐', '喜', '狂', '厉', '冷', '狠'];

  const totalChars = countChineseChars(text);
  if (totalChars === 0) return 0;

  let signalCount = 0;
  for (const word of [...actionWords, ...emotionWords]) {
    const matches = text.match(new RegExp(word, 'g'));
    if (matches) signalCount += matches.length;
  }

  return signalCount / (totalChars / 100); // 每 100 字的信号词密度
}

// 检查章末钩子
function checkEndingHook(text) {
  const lastChars = text.slice(-400);

  // 钩子信号
  const hookSignals = [
    '？', '?',          // 问句
    '……',              // 省略（悬念）
    '突然', '忽然',     // 突发
    '没想到', '不料',   // 意外
    '但是', '可是',     // 转折
    '来了', '到了',     // 即将到来
    '嘴角', '笑了笑',   // 意味深长的表情
    '不知道', '该死',   // 紧张感
  ];

  return hookSignals.some(signal => lastChars.includes(signal));
}
```
