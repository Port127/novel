## 二、6 项架构加固

### 2.1 名字解析协议（ADR-10）

**协议：** `_protocols/name-resolution.md`

- 称呼选择规则：按 POV 与角色关系亲疏决定用哪个称呼
- 项目级命名配置：`meta.yaml` 新增 `naming` 块（era/culture/forbidden_patterns）
- 重命名事务化：`dry-run → confirm → execute`，预览全部受影响文件

**影响 Skill：** `character-add`（起名参照配置）、`character-edit`（改名走事务流程）、`chapter-draft` / `anti-ai-rewrite`（遵守称呼规则）

### 2.2 预检完整性协议（ADR-11）

**协议：** `_protocols/preflight-integrity.md`

- 关键操作前验证引用链（目标章节 ± 2 章范围）
- 检测孤儿索引/文件、版本文件缺失、前序章节 summary 缺失
- `novel-doctor` 新增 `--quick` 模式（仅检查当前章节周边）

**影响 Skill：** `pipeline-chapter-kickoff`、`chapter-draft`、`chapter-update`、`pipeline-draft-polish`

### 2.3 大纲分层与粒度检测（ADR-12）

- `plot-add` 根据内容具体程度分流：场景级 → 章节文件，幕级 → outline.md
- 粒度不匹配时主动提示用户选择

### 2.4 风格生命周期管理（ADR-13）

**协议：** `_protocols/style-lifecycle.md`

- 三阶段：积累期（0-2 章不触发）→ 提炼触发（≥ 3 章主动发起）→ 漂移检测（每 10 章提醒更新）
- `chapter-update` 触发提炼，`pipeline-draft-polish` 检测漂移

### 2.5 素材消化结构化输出（ADR-14）

- `ingestion_brief.md` 改为 YAML front matter + Markdown body
- 每条提取内容附置信度标记：`verbatim`（原文）/ `inferred`（推理）/ `supplemented`（补充）
- 下游按置信度分级处理：verbatim 直接用，inferred 标注，supplemented 必须确认

### 2.6 操作日志与来源水印（ADR-15）

**协议：** `_protocols/operation-journal.md`

- 多文件写入前后记录 `ops_log.yaml`，启动时检测未完成操作
- `chapter-draft` 追加"内容来源追溯表"（哪些来自大纲/角色卡/AI 补充）

## 三、3 项功能扩展

### 3.1 钩子/伏笔系统（ADR-6）

**新增 Skill：**
| Skill | 功能 |
|-------|------|
| `/hook-add` | 登记伏笔，分三级（major/minor/micro），设回收截止和条件 |
| `/hook-query` | 查询伏笔，按状态/级别/章节/逾期过滤，含时间轴视图 |
| `/hook-resolve` | 回收/放弃/延期伏笔 |

**联动增强：**
- `pipeline-chapter-kickoff`：开写时自动提醒近期到期钩子
- `chapter-review`：按级别检查伏笔密度
- `consistency-check`：纳入钩子健康度检查

### 3.2 设定演化与有效期（ADR-7）

- 设定模板新增字段：`valid_from` / `valid_until` / `expiry_trigger` / `supersedes` / `superseded_by`
- `/setting-add --supersedes`：创建接替设定并自动废弃旧版
- `/setting-edit --evolve`：以旧设定为基础演化新版
- `chapter-draft` 生成时自动过滤过期设定
- `pipeline-chapter-kickoff` 开写时提醒即将过期设定

### 3.3 多版本章节草稿（ADR-8）

**新增 Skill：**
| Skill | 功能 |
|-------|------|
| `/chapter-compare` | 对比多个版本的差异（结构/节奏/角色焦点） |

**增强：**
- `/chapter-draft --alt [备注]`：生成备选版本（`ch001_v2.md`）
- `/chapter-update --promote v{N}`：提升备选为主版本
- `chapters/index.yaml` 新增 `versions` 数组和 `active_version`
