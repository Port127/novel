# 任务

对指定范围内的章节进行文风一致性审查，检测风格漂移、质量波动和异常章节。

## 前置检查

1. 读取 `.current.yaml` 获取 `current_path`
2. 读取 `{current_path}/chapters/index.yaml` 获取章节列表
3. 若指定了范围，筛选对应章节；未指定则取全部 `draft` 及以上状态的章节
4. 若存在 `{current_path}/.novel/meta.yaml` 中的 `style` 配置，作为风格基准

## 输入参数

- `$0` (range): 可选章节范围，如 `ch001-ch010`、`全部`
- `--baseline [章节ID]`: 可选，指定某一章作为风格基准（默认取最近的 `final` 章节）
- `--detail`: 输出逐章详情而非仅汇总

## 执行步骤

### 1. 抽样与提取

- 若范围内章节 ≤ 10，全量读取
- 若 > 10 章，均匀抽样 8-10 章（首章、末章必选，中间均匀分布）
- 从每章中提取：
  - 叙述段落样本（非对白，300-500 字）
  - 对白段落样本（200-300 字）
  - 首段和末段

### 2. 多维度文风分析

对每章样本进行以下维度评分（1-5 分）：

| 维度 | 检查内容 |
|------|---------|
| 句式节奏 | 长短句比例、句式变化度、是否趋向单一模式 |
| 描写密度 | 叙述与描写的比例、是否某些章过度描写而另一些过于干瘪 |
| 比喻频率 | 比喻/修辞的密度是否前后差异过大 |
| 对白风格 | 对白的口语化程度、是否前面章节口语化后面变正式（或反之）|
| 叙事距离 | 视角的远近——是否有些章贴近角色内心，有些章突然变成全知上帝视角 |
| 情绪基调 | 整体情绪色调是否与故事阶段匹配（不是要求统一，而是检查是否有不合理跳变）|

### 3. 检测风格漂移

- **渐变漂移**：前几章和后几章在某个维度上呈现明显的单向变化趋势（如越写越啰嗦、越写对白越书面化）
- **突变异常**：某一章在多个维度上与前后章显著不同（可能是不同状态下写的，或被 AI 改写过）
- **AI 混入嫌疑**：若某章的比喻密度、描写占比、过渡词使用突然飙升，标记为"建议用 `/anti-ai-check` 单独检测"

### 4. 输出报告

## 输出格式

```
🎨 文风一致性审查

范围：{{range}}
抽样：{{sample_count}} 章
基准章节：{{baseline}}

## 整体判断：{{overall}}

## 维度一致性
| 维度 | 均值 | 波动 | 趋势 | 异常章 |
|------|------|------|------|--------|
| 句式节奏 | 3.5 | ±0.5 | 稳定 | - |
| 描写密度 | 4.0 | ±1.5 | ↑递增 | ch008 |
| 比喻频率 | 2.5 | ±0.3 | 稳定 | - |
| 对白风格 | 3.0 | ±1.0 | ↓递减 | ch005, ch009 |
| 叙事距离 | 3.5 | ±0.5 | 稳定 | - |
| 情绪基调 | - | - | 符合剧情 | - |

## 风格漂移警告
- ⚠️ 描写密度：ch001-ch003 平均 2.5，ch008-ch010 平均 4.5，呈递增趋势——后期可能过度描写
- ⚠️ 对白风格：ch005 和 ch009 对白书面化程度显著高于其他章节

## 异常章节
- ch008：比喻密度和描写占比同时飙升，建议 `/anti-ai-check ch008`

## 建议
1. ...
2. ...
```

## 注意事项

- 文风"不一致"不一定是坏事——不同情节阶段可能需要不同的叙事节奏
- 审查重点是"无意识的漂移"和"突变异常"，而非要求每章文风完全相同
- 维度评分为相对值，用于章间比较，不作为绝对质量判断
- 若项目中设置了 `style.template`，额外对比模板特征
