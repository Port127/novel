# 修订设定

修订已有设定，检查与其他设定的冲突。

## 前提条件

设定已存在（非空模板）。

## 工作流程

### 1. 确认项目

运行 `python scripts/project.py list` 查看项目列表。如果用户未指定项目，询问选择。

### 2. 选择设定类型

| 类型 | 对应文件 | 前置依赖 |
|------|---------|---------|
| 世界观 | worldbuilding.yaml | 无 |
| 人物 | characters.yaml | 世界观完善 |
| 大纲 | outline.yaml | 世界观+人物完善 |

### 3. 展示当前设定

读取设定文件，展示结构摘要：

- **世界观**：力量体系、势力列表、地点列表
- **人物**：人物列表、主角特征、关键关系
- **大纲**：premise、幕数、序列数、节拍数

### 4. 询问修订内容

用户描述修订方向：

| 类型 | 典型修订 |
|------|---------|
| 世界观 | 添加势力、修改力量等级、添加地点 |
| 人物 | 添加角色、修改关系、调整弧线 |
| 大纲 | 调整节拍、合并序列、修改张力 |

### 5. 冲突检查

检查修订是否与其他设定冲突：

| 修订类型 | 冲突检查点 |
|---------|-----------|
| 世界观·势力 | 人物是否关联该势力？大纲是否引用该势力？|
| 世界观·力量等级 | 人物等级是否匹配？大纲是否使用该等级？|
| 人物·关系 | 大纲节拍是否引用该关系？|
| 人物·弧线 | 大纲转折点是否匹配弧线？|
| 大纲·节拍 | 人物出场是否匹配？势力行动是否匹配？|

**冲突检查方法**：

```bash
# 检查势力引用
grep -r "{势力名}" novels/{project_id}/settings/characters.yaml
grep -r "{势力名}" novels/{project_id}/settings/outline.yaml

# 检查人物引用
grep -r "{人物名}" novels/{project_id}/settings/outline.yaml

# 检查等级引用
grep -r "{等级名}" novels/{project_id}/settings/*.yaml
```

### 6. 冲突处理

| 冲突状态 | 处理 |
|---------|------|
| 无冲突 | 直接修订 |
| 有冲突 | 展示冲突点，询问用户选择 |

**冲突处理选项**：

```
发现冲突：
  - 人物 {A} 关联势力 {B}
  - 大纲节拍 12 引用势力 {B}

请选择：
  1. 同步修订（势力 {B} → {C}，同时更新人物和大纲）
  2. 取消修订（保留原设定）
  3. 仅修订世界观（后续手动处理冲突）
```

### 7. 完善度重新检查

修订后重新计算完善度：

```bash
python scripts/utils/completeness_check.py {project_id} {setting_type}
```

确保修订后仍达到阈值，否则提示用户补充缺失字段。

### 8. 更新状态

修订完成后，更新 project.yaml 的 `updated` 字段。

## 冲突检查清单

| 检查项 | 说明 |
|-------|------|
| 势力引用一致性 | 势力改名/删除 → 检查人物和大纲引用 |
| 人物引用一致性 | 人物改名/删除 → 检查大纲节拍引用 |
| 力量等级一致性 | 等级改名/调整 → 检查人物等级字段 |
| 弧线转折一致性 | 人物弧线调整 → 检查大纲转折点匹配 |
| 节拍人物匹配 | 大纲节拍调整 → 检查出场人物是否在人物列表 |

## 参考

- 完善度标准：`data/schemas/completeness.schema.yaml`
- 设定 Schema：`data/schemas/{setting_type}.schema.yaml`
- 注意：修订是迭代的起点，确保每次修订都检查冲突
