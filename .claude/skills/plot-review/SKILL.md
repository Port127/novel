# 任务

对现有大纲进行结构化审查，并给出可直接落地到 `plot/outline.md` 与 `plot/outline.yaml` 的优化建议。

## 前置检查

1. 读取 `.current.yaml` 获取 `current_path`
2. 读取 `{current_path}/plot/outline.md`
3. 若存在，读取 `{current_path}/plot/outline.yaml`
4. 若存在，读取 `{current_path}/chapters/index.yaml` 了解大纲与章节推进是否脱节

## 输入参数

- `$0+` (focus): 可选优化重点，如“中点太弱”“后半段松散”“伏笔不够”

## 执行步骤

### 1. 提取当前剧情骨架

识别以下信息：

- premise / 主线目标是否清晰
- 幕次或章节结构是否完整
- 主冲突、次冲突、关键转折是否可定位
- 主角目标、阻碍与代价是否连成因果链
- 关键钩子是靠异常细节还是靠硬拐
- 伏笔埋设与回收是否形成链路

### 2. 审查结构质量

**像编剧看剧本，不像老师改作业：**
- 不只看"有没有中点转折"，要看"这个转折是否改变了读者对前面所有信息的理解"
- 不只看"有没有代价"，要看"这个代价是否让角色和读者都觉得值得但心痛"
- 钩子的终极检验：如果你是读者，读到这里会不会花钱买下一章？

重点检查：

- 开场是否尽快建立人物处境与核心问题
- 第一推动事件是否足够改变主角状态
- 中段是否存在升级与中点转折
- 后段是否存在危机抬升与最终收束
- 爽点是否来自信息差、下注与代价，而不是单纯平推
- 幕次占比与章节分布是否失衡
- 关键桥段是否过于陈旧，缺少立场、顺序、代价或结果上的新变化

### 3. 审查节奏与兑现

重点检查：

- 是否存在连续“说明而非推进”的区段
- 高张力节点是否分布过密或过 sparse
- 钩子是否埋在谎言、口误、巧合或异常意象中，而不是只靠硬性反转
- 伏笔是否只埋不收，或回收缺桥接
- 悲剧 / BE 是否只是“惨”，还是建立在错过、误判、等待落空之上
- 已创建章节是否偏离当前大纲

### 4. 输出优化动作

给出 3-6 条按优先级排序的动作，要求：

- 尽量具体到幕次、章节段或关键事件
- 说明“为什么要改”
- 说明“改完后解决什么问题”
- 优先指出哪里该补“代价”“异常钩子”“信息差”或“老套路魔改”

## 输出格式

```
🧭 大纲审查

当前判断：{{overall_status}}
关注重点：{{focus}}

✅ 已成形
- ...

⚠️ 主要问题
- ...

🔧 优化动作（按优先级）
1. ...
2. ...
3. ...

建议下一步：
- /pipeline-outline-polish {{focus}}
- /plot-add {{chapter_or_node}} {{content}}
- /plot-suggest {{problem}}
```

## 注意事项

- 审查优先指出“结构性问题”，不是润色句子
- 不直接重写整份大纲，优先给出可执行补强动作
- 若需要大规模重排幕次，先给变更预览再执行
- 不把“更惨”误判为“更有冲击力”，优先看错过、代价和兑现
