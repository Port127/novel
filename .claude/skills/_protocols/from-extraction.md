# 引用提取协议（--from）

当 skill 支持 `--from` 参数时，按以下统一流程处理。

## 1. 判断输入来源

- 如果 `--from` 是文件路径（如 `chapters/ch003.md`、`drafts/notes.md`），读取文件内容
- 如果 `--from` 后面是引号包裹的文本，直接使用该文本
- 如果用户通过 IDE 选中了文本并触发命令，将选中内容作为输入

## 2. 智能提取

按调用 skill 的类型提取对应信息：

| 调用 skill | 提取目标 | 提取方式 |
|-----------|---------|---------|
| `character-add` | 人物 | 有姓名时：用参考内容填充单人卡。无姓名时：扫描所有人物，逐个提取 |
| `plot-add` | 情节点 | 识别关键事件、冲突点、转折；推断章节位置、出场人物、冲突与钩子 |
| `setting-add` | 设定要素 | 识别规则、势力、地理、术语；推断 category 和 summary |
| `relationship-add` | 角色关系 | 扫描角色间互动、称呼、态度；推断关系类型、张力来源和动态机制 |

每个独立单元拆为一条创建请求。`source` 标注引用文件路径和行号。

### 来源边界规则（强制）

**每一条提取内容必须能指向源文件的具体行号。无法指向则不得列入提取结果。**

提取时，对每条内容打标签：

| 标签 | 机读值 | 含义 | 处理方式 |
|---|---|---|---|
| `✅ 原文` | `verbatim` | 源文件中有明确对应内容，附行号 | 正常列入 |
| `🔶 推断` | `inferred` | 源文件中无原文，但可从上下文合理推断 | 列入，但必须标注，用户确认后才写入 |
| `❌ 非来源` | `supplemented` | 来自大纲、人物卡、设定等其他项目材料，或纯 AI 补充 | **不得列入提取结果**，禁止混入 |

**机读格式**（用于 `ingestion_brief.md` 等结构化输出）：

```markdown
<!-- confidence: verbatim, source_lines: [3, 5, 7] -->
```

每个提取段落/条目的上方用 HTML 注释附带 `confidence` 和 `source_lines`，供下游 skill 机读。

**禁止行为：**
- 把从 `outline.md` 读来的内容混进"从草稿提取的情节点"
- 把从 `characters/*.yaml` 读来的设定混进"从草稿提取的角色信息"
- 用 `inferred` 标签掩盖实际上来自其他项目材料的内容（应标 `supplemented`）

如果确实需要参考其他项目材料来补充语境，在输出末尾**单独列出**，明确注明"以下来自 [文件名]，非本次源文件内容"。

## 3. 用户确认

- 列出所有识别到的条目，每条附来源标签和行号
- 用户逐个确认、修改或跳过
- 带 `🔶 推断` 标签的条目必须等用户确认，不得跳过
- 确认后进入该 skill 的标准创建流程

## 4. 与 --quick 组合

`--from ... --quick` 会提取后直接按快捕模式写入，不逐条确认。

## 边界

若输入内容同时包含多种类型（设定 + 角色 + 剧情混杂），应建议用户改用 `/pipeline-note-triage` 统一分拣。

## 引用方式

在 skill 中写：

```
当指定 `--from` 时，按 [引用提取协议](_protocols/from-extraction.md) 执行提取与确认流程。
```

## 引用此协议的 skill

- `character-add`
- `plot-add`
- `setting-add`
- `relationship-add`
