# 任务

检查指定角色在章节范围内的对白一致性与辨识度。

## 前置检查

1. 读取 `.current.yaml` 获取 `current_path`
2. **角色自动推断**：如果 `$0`（角色名）未提供——检查用户当前打开/引用的文件是否为 `characters/*.yaml`，若是则提取角色名；否则提示用户指定
3. **章节范围自动推断**：如果 `$1`（范围）未提供——按 [章节自动推断协议](_protocols/chapter-auto-inference.md) 确定检查范围
4. 读取角色卡 `{current_path}/characters/$0.yaml`
5. 读取章节内容（按 `$1` 范围）

## 输入参数

- `$0` (character): 角色名
- `$1` (range): 章节范围，如 `ch001-ch010`（可选）

## 执行步骤

### 1. 提取对白样本

抽取该角色对白，统计常用句法、语气词、口头禅。

### 2. 对比角色设定

检查是否符合角色卡中的性格和身份。

### 3. 对比其他角色

检测与其他主角对白风格是否过度重合。

## 输出格式

```
🎙️ 人物声音检查：$0

一致性：{{consistency}}
区分度：{{distinctiveness}}

问题：
- ...

建议：
- 增加专属口头禅/句式
- 调整语气强弱与信息密度
```

## 注意事项

- 建议同时检查 2-3 个关键角色做横向对比
- 不以“口癖堆砌”替代人物声音
