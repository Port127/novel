# 任务

查看时间线事件。

## 前置检查

1. 读取 `.current.yaml` 获取 `current_path`

## 输入参数

- `$0` (range): 可选范围，如 `第5-10天`、`第1章-第5章`

支持的筛选：
- 时间范围：`第5-10天`
- 章节范围：`--chapter 1-5`
- 角色筛选：`--character 张三`
- 地点筛选：`--location 青云宗`
- `--multi-thread`: 多线并行视图模式（见下方说明）

## 执行步骤

### 1. 读取时间线

读取 `{current_path}/timeline/main.yaml`。

### 2. 应用筛选

根据参数过滤事件。

### 3. 多线并行视图（--multi-thread 模式）

当指定 `--multi-thread` 时：

1. **识别叙事线**
   - 从时间线事件中提取参与角色，按角色或角色组分为独立叙事线
   - 也可从 `plot/outline.md` 的结构中识别显式标注的多线（如"主线""支线A"等）

2. **生成并行视图**
   - 按时间轴横向排列，每条叙事线作为一列
   - 标记 **交汇点**（多条线在同一时间/事件相遇）和 **分离点**（角色分开行动）
   - 标记 **信息差**：某条线的角色知道而另一条线的角色不知道的信息

3. **输出诊断**
   - 两条线是否有"太久没交汇"的风险（读者可能遗忘副线角色）
   - 交汇点处是否有足够的戏剧冲突
   - 并行时间是否对齐（A 线过了 3 天而 B 线过了 1 天？）

### 4. 格式化输出

## 输出格式

```
📅 时间线：{{范围}}

---

## 第5天
- 🌅 清晨：张三离开山村 → 第3章
- 🌞 上午：张三抵达青云宗 → 第4章
- 🌙 夜晚：张三通过入门测试 → 第5章

## 第6天
- 🌞 全天：张三开始修炼基础功法 → 第6章
- 🌙 夜晚：张三遇见赵六 → 第7章

---

📊 统计：
   时间范围：{{start}} ~ {{end}}
   事件数量：{{count}}
   涉及角色：{{characters}}

💡 添加事件：/timeline-add [时间] [事件]
```

**多线并行视图（--multi-thread 模式）：**

```
📅 多线并行视图：{{范围}}

        | 陈默线        | 宋行舟线      | 白烬会线
--------|---------------|---------------|---------------
第1天   | 发现灰盒 ch01 |               |
第2天   | 新手试炼 ch02 | 废墟搜索 ch03 |
第3天   | ⚡交汇：废墟遭遇 ch04          | 远程监控
第4天   | 据点建设 ch05 |               | ← 信息差：不知据点位置
第5天   |               | 独自侦查 ch06 | 部署追踪 ch07
第6天   | ⚡交汇：伏击事件 ch08           |

🔍 诊断：
- 白烬会线在第1-2天无事件，建议补 1 个铺垫事件
- 第3天交汇有冲突支撑（✅）
- 宋行舟线第4-5天独立时长合理
```

## 注意事项

- 时间格式统一
- 显示章节关联
- 支持多维度筛选
- `--multi-thread` 模式下，叙事线的划分优先用角色组，若 `outline.md` 有显式标注则用标注
- 交汇点处的冲突质量比交汇频率更重要——不用每天都交汇，但每次交汇要有事发生
