# 任务

添加情节节点到大纲。

## 前置检查

1. 读取 `.current.yaml` 获取 `current_path`

## 输入参数

- `$0` (target): 目标标识。格式决定写入位置：
  - `ch001`、`ch012` 等章节 ID → **章节模式**，写入对应章节文件
  - `第一幕`、`第二季`、`arc-*` 等幕/弧线标识 → **弧线模式**，写入 `outline.md`
- `$1+` (content): 情节内容描述
- `--from`: 从文件或引用内容中提取情节点（见步骤 0）
- `--arc`: 强制切换到弧线模式（即使 $0 看起来像章节 ID）

## 执行步骤

### 0. 引用模式（--from）

当指定 `--from` 时，按 [引用提取协议](_protocols/from-extraction.md) 执行提取与确认流程。

提取重点：识别关键事件、冲突点、转折；推断章节位置、出场人物、冲突与钩子。每个独立事件拆为一个情节节点。

### 1. 判断写入模式（含粒度检测）

根据 `$0` 格式和内容粒度综合判断：

**步骤 A：目标识别**

| 条件 | 初步模式 |
|---|---|
| `$0` 匹配 `ch\d+` 格式，或 `chapters/index.yaml` 中存在该 ID | 章节模式 |
| `$0` 包含"幕"/"季"/"arc"/"Part"等关键词，或使用 `--arc` 标志 | 弧线模式 |
| 无法判断 | 询问用户确认 |

**步骤 B：内容粒度检测（防止场景级内容误入 outline.md）**

对 `$1+` 的内容进行粒度分析：

| 粒度信号 | 判定 | 正确去处 |
|----------|------|---------|
| 含具体动作/对白描述（如"主角坐在宿舍打游戏"） | 场景级 | 章节文件 `## 场景大纲` |
| 提到具体章节号且含事件细节 | 章节级 | 章节文件 `## 场景大纲` |
| 只有抽象目标/转折/弧线方向（如"主角从防守转为进攻"） | 幕级 | `outline.md` |
| 含多个阶段、多章跨度（如"发现→对峙→逃离→反思"） | 多章弧线 | `outline.md`（弧线节点），再拆到各章 |

**粒度冲突处理**：

- 弧线模式但内容是场景级 → 提示：
  ```
  💡 这条内容看起来是具体场景/事件，更适合写入章节文件的场景大纲。
     要写入 chapters/{推断的章节}.md 吗？还是作为弧线级概述写入 outline.md？
  ```
- 章节模式但内容是幕级 → 正常写入章节文件，同时提示：
  ```
  💡 这条内容比较抽象，也可以作为弧线节点写入 outline.md（/plot-add {幕标识} {内容} --arc）
  ```

### 2. 章节模式：写入章节文件

目标文件：`{current_path}/chapters/{chapter_id}.md` 的 `## 场景大纲` 区。

若文件不存在，提示先执行 `/chapter-create {chapter_id}`。

**情节描述不是会议纪要，是故事的呼吸：**
- "事件"字段用有画面感的语言，不用"主角得知消息"，而用"主角从一封被烧了一半的信里拼出真相"
- "冲突"字段写出角色内心的撕裂，不只是外部障碍
- "钩子"字段要让读到大纲的人也想知道后面发生了什么

写入格式：

```markdown
## 场景大纲

**时间**：{时间线位置}
**地点**：{场景地点}
**人物**：{出场人物}
**事件**：{有画面感的情节描述}
**冲突**：{角色内心撕裂}
**钩子**：{结尾悬念}
```

### 3. 弧线模式：写入 outline.md

读取 `{current_path}/plot/outline.md`，根据 `$0` 确定插入位置：
- `第一幕` → 插入到对应幕节点
- `--after 第二季` → 在指定节点后插入

写入格式（幕/弧线级，不含逐场细节）：

```markdown
### {幕/弧线标题}

**走向**：{本幕核心走向}
**主线拐点**：{关键转折}
**关联章节**：{如 ch001-ch015}
**悬而未决**：{留给下一幕的钩子}
```

### 4. 更新状态

更新 `{current_path}/.novel/state.yaml`：
- `project.updated`：今天日期

## 输出格式

章节模式：
```
✅ 场景大纲已写入章节文件

📖 章节：{chapter_id}
📄 已更新：chapters/{chapter_id}.md → ## 场景大纲

💡 下一步：/chapter-draft {chapter_id}
```

弧线模式：
```
✅ 弧线节点已添加

📖 位置：{幕/弧线标识}
📄 已更新：plot/outline.md

💡 继续添加：/plot-add {下一幕} {内容} --arc
```

## 注意事项

- **分层原则**：`outline.md` 只放幕/弧线级节点（全书导航），章节级场景/冲突/钩子只写章节文件
- 内容粒度不匹配时主动提示用户选择正确去处，不默认塞入 outline.md
- 弧线模式下注明关联章节 ID，便于后续追溯
- 支持多故事线，通过 `--arc` 标志区分
