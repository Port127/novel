# 任务

添加情节节点到大纲。

## 前置检查

1. 读取 `.current.yaml` 获取 `current_path`

## 输入参数

- `$0` (chapter): 章节标识，如 `第1章`、`第一幕.开场`
- `$1+` (content): 情节内容描述
- `--from`: 从文件或引用内容中提取情节点（见步骤 0）

## 执行步骤

### 0. 引用模式（--from）

当指定 `--from` 时，按 [引用提取协议](_protocols/from-extraction.md) 执行提取与确认流程。

提取重点：识别关键事件、冲突点、转折；推断章节位置、出场人物、冲突与钩子。每个独立事件拆为一个情节节点。

### 1. 读取大纲

读取 `{current_path}/plot/outline.md`。

### 2. 解析位置

根据 `$0` 确定插入位置：
- `第5章` → 插入到第五章位置
- `第一幕.开场` → 插入到指定节点
- `--after 第4章` → 在第4章后插入

### 3. 添加情节节点

**情节描述不是会议纪要，是故事的呼吸：**
- "事件"字段用有画面感的语言，不用"主角得知消息"，而用"主角从一封被烧了一半的信里拼出真相"
- "冲突"字段写出角色内心的撕裂，不只是外部障碍
- "转折"字段要让读到大纲的人也想知道后面发生了什么

按标准格式添加：

```markdown
### 第X章：{标题}

**时间**：{时间线位置}
**地点**：{场景地点}
**人物**：{出场人物}
**事件**：{情节描述}
**冲突**：{本章冲突点}
**转折**：{章节结尾钩子}
```

### 4. 更新状态

更新 `{current_path}/.novel/state.yaml`：
- `project.updated`：今天日期

## 输出格式

```
✅ 情节节点已添加

📖 位置：$0
📝 内容：$1

📄 已更新：plot/outline.md

💡 继续添加：/plot-add [下一章] [内容]
```

## 注意事项

- 章节号自动递增
- 可指定插入位置
- 支持多故事线
