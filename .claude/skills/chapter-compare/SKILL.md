# 任务

对比同一章节的两个草稿版本，输出多维度分析。

## 前置检查

1. 读取 `.current.yaml` 获取 `current_path`
2. 读取 `{current_path}/chapters/index.yaml`，检查该章节是否存在 `versions` 字段
3. 若无 `versions`，扫描目录中的 `{chapter_id}_v*.md` 文件自动发现版本

## 输入参数

- `$0` (chapter_id): 章节 ID（必需）
- `$1` (version_a): 版本标识，如 `v1`（可选，默认主稿）
- `$2` (version_b): 版本标识，如 `v2`（可选，默认最新备选）
- `--all`: 对比所有版本（输出矩阵式概览）

## 执行步骤

### 1. 定位版本文件

| 版本标识 | 对应文件 |
|---|---|
| `v1` 或省略 | `chapters/{chapter_id}.md`（主稿） |
| `v2` | `chapters/{chapter_id}_v2.md` |
| `v{N}` | `chapters/{chapter_id}_v{N}.md` |

若指定版本文件不存在，提示可用版本列表。

### 2. 读取两个版本

分别读取两个版本的正文内容（`## 正文草稿` 区域），跳过元数据和场景大纲。

### 3. 多维度对比

#### 基础指标

| 指标 | 版本 A | 版本 B |
|---|---|---|
| 字数 | | |
| 场景数 | | |
| 对白占比 | | |
| 描写占比 | | |

#### 结构对比

- 开场方式差异：各版本从什么角度/事件切入
- 场景序列差异：事件排列顺序是否不同
- 结尾钩子差异：各版本留下的悬念是什么
- 节奏曲线差异：哪个版本节奏更紧凑/更有呼吸感

#### 风格对比

- 叙事视距：哪个更贴近角色内心，哪个更客观
- 语言风格：句式长短、修辞密度、对白风格
- 情绪基调：整体氛围的差异

#### 各版本独到之处

- 版本 A 的亮点：这个版本做到了什么是另一个版本没有的
- 版本 B 的亮点：同上

### 4. 合并建议（可选）

如果两个版本各有优势，给出具体的合并建议：
- "版本 A 的开场 + 版本 B 的中段冲突 + 版本 A 的结尾"
- 标注哪些段落可以直接复用，哪些需要过渡衔接

## 输出格式

```
🔀 章节版本对比：{{chapter_id}}

📊 基础指标
| 指标 | {{version_a}} | {{version_b}} |
|---|---|---|
| 字数 | {{N}} | {{N}} |
| 场景数 | {{N}} | {{N}} |
| 对白占比 | {{N}}% | {{N}}% |

📖 结构差异
- 开场：A {{描述}} / B {{描述}}
- 结尾：A {{描述}} / B {{描述}}
- 节奏：{{对比描述}}

✨ 各版本亮点
  版本 A：{{亮点}}
  版本 B：{{亮点}}

🔧 合并建议
  {{具体建议}}

下一步：
   /chapter-update {{chapter_id}} --promote v2  将版本 B 设为主稿
   直接编辑 chapters/{{chapter_id}}.md          手动合并
```

### 全版本概览模式（--all）

```
🔀 章节版本概览：{{chapter_id}}（{{N}} 个版本）

| 版本 | 字数 | 开场方式 | 结尾钩子 | 独到之处 |
|---|---|---|---|---|
| v1（主稿） | {{N}} | {{摘要}} | {{摘要}} | {{一句话}} |
| v2 | {{N}} | {{摘要}} | {{摘要}} | {{一句话}} |
| v3 | {{N}} | {{摘要}} | {{摘要}} | {{一句话}} |

```

## 注意事项

- 对比是只读操作，不修改任何文件
- 不打分不排名，分析各版本的独到价值
- 合并建议是可选参考，不强推某个方案
- 如果只有一个版本，提示"只找到主稿，使用 `/chapter-draft --alt` 生成备选版本"
- 对白对比时注意角色声音区分度——哪个版本的角色声音更鲜明
