# 创作流程管理

统一创作入口，管理整个写作流程。

## 核心原则

**小说应该"生长和演进"，而非"一次性生成"。**

```
世界观（根基）
    ↓ 生长
人物（在世界观中生根）
    ↓ 生长
大纲（在人物弧线上生长）
    ↓ 生长
章节规划（在大纲中细化）
    ↓ 生长
正文（在所有设定基础上写作）
```

---

## 流程阶段

| 阶段 | 设定类型 | 前置依赖 | 完善度阈值 | 对应 Skill |
|------|---------|---------|-----------|-----------|
| 1 | 世界观 | 无 | 80% | nm → 交互生成 |
| 2 | 人物 | 世界观完善 | 70% | generate-character |
| 3 | 大纲 | 世界观+人物完善 | 85% | generate-outline |
| 4 | 章节规划 | 大纲完善 | 100% | generate-chapter |
| 5 | 正文 | 章节规划存在 | - | write-chapter |

---

## 工作流程

### 1. 项目状态检查

读取 `project.yaml` 状态字段：

| 状态 | 可执行阶段 | 说明 |
|------|-----------|------|
| planning | 1-4 | 规划阶段，可生成设定和章纲 |
| drafting | 5 | 写作阶段，可写正文 |
| revising | 修订 | 可使用 revise-setting |
| completed | 导出 | 只可导出 |

### 2. 依赖状态检查

检查当前阶段前置依赖是否完善：

```bash
python scripts/utils/completeness_check.py {project_id} all
```

检查结果解读：

| 检查项 | 阈值 | 意义 |
|-------|------|------|
| worldbuilding | 80% | 世界观足够支撑人物和大纲 |
| characters | 70% | 人物足够支撑大纲 |
| outline | 85% | 大纲足够支撑章节规划 |

### 3. 流程引导

根据依赖状态引导用户：

**场景A：新项目，无任何设定**

```
当前状态：无设定
建议：开始阶段1 - 世界观设定

执行步骤：
  1. 使用 nm 检索同类题材素材参考
  2. 交互式询问：力量体系 → 势力 → 地点 → 规则
  3. 生成 worldbuilding.yaml
  4. 完善度检查：确认达到 80%

是否开始世界观设定？
```

**场景B：世界观完善，人物未完善**

```
当前状态：
  - 世界观完善度 85% ✅
  - 人物完善度 0% ❌
  - 大纲完善度 0% ❌

建议：开始阶段2 - 人物设定

执行步骤：
  1. 基于世界观中的势力、力量等级设计人物
  2. 交互式询问：主角 → 反派 → 关键配角 → 关系
  3. 生成 characters.yaml
  4. 完善度检查：确认达到 70%

是否开始人物设定？
```

**场景C：世界观+人物完善，大纲未完善**

```
当前状态：
  - 世界观完善度 85% ✅
  - 人物完善度 75% ✅
  - 大纲完善度 0% ❌

建议：开始阶段3 - 大纲设定

执行步骤：
  1. 基于人物弧线设计大纲转折点
  2. 交互式询问：premise → 幕 → 序列 → 节拍
  3. 生成 outline.yaml
  4. 完善度检查：确认达到 85%

是否开始大纲设定？
```

**场景D：所有设定完善**

```
当前状态：
  - 世界观完善度 85% ✅
  - 人物完善度 75% ✅
  - 大纲完善度 90% ✅

建议：开始阶段4 - 章节规划

执行步骤：
  1. 转化大纲为章节摘要
  2. 分配张力值
  3. 生成 chapters/_index.yaml

是否开始章节规划？
```

**场景E：章节规划完成**

```
当前状态：
  - 世界观完善度 85% ✅
  - 人物完善度 75% ✅
  - 大纲完善度 90% ✅
  - 章节规划完成 ✅

建议：开始阶段5 - 正文写作

执行步骤：
  1. 选择要写的章节
  2. 确认章节摘要和上下文
  3. 生成正文
  4. 续写/改写选项

是否开始正文写作？可使用 /write-chapter
```

### 4. 跳阶段阻止

用户尝试跳阶段时阻止并提示：

```
用户尝试：/generate-outline

检查结果：
  - worldbuilding 完善度 20% ❌ (需 ≥ 80%)
  - characters 完善度 0% ❌ (需 ≥ 70%)

阻止原因：
  - 世界观不完善，缺失字段：power_system.name, factions, locations
  - 人物列表为空

引导：
  请先完成世界观设定（当前完善度 20%）
  是否开始世界观设定？
```

### 5. 修订入口

用户想修订设定时：

```
用户请求：修订势力设定

执行：
  1. 引导使用 /revise-setting
  2. 修订后重新检查依赖状态
  3. 提示可能的冲突处理
```

## 前置依赖检查（强制）

**generate-outline 必须检查**：

```bash
python scripts/utils/completeness_check.py {project_id} worldbuilding
python scripts/utils/completeness_check.py {project_id} characters
```

| 状态 | 处理 |
|------|------|
| worldbuilding ≥ 80%, characters ≥ 70% | 允许继续 |
| 不满足 | 阻止并引导先完成前置设定 |

**generate-character 必须检查**：

```bash
python scripts/utils/completeness_check.py {project_id} worldbuilding
```

| 状态 | 处理 |
|------|------|
| worldbuilding ≥ 80% | 允许继续 |
| 不满足 | 阻止并引导先完成世界观 |

## 交互式询问流程

### 世界观生成询问

```
1. 确认题材类型（修仙/都市/科幻/悬疑等）
   ↓
2. 询问力量体系（名称、等级、晋升规则）
   ↓
3. 询问势力关系（主要势力、冲突关系）
   ↓
4. 询问地理空间（世界范围、重要地点）
   ↓
5. 询问核心规则（世界运行的底层逻辑）
   ↓
6. 汇总确认 → 生成 YAML
```

### 人物生成询问

```
1. 基于世界观确认主角设定
   ↓
2. 询问反派（动机、与主角冲突）
   ↓
3. 询问关键配角（功能定位、人物弧线）
   ↓
4. 询问人物关系（核心关系、张力来源）
   ↓
5. 汇总确认 → 生成 YAML
```

### 大纲生成询问

```
1. 确认 premise（一句话概括故事）
   ↓
2. 基于人物弧线设计幕结构
   ↓
3. 设计序列（每幕的推进节奏）
   ↓
4. 设计节拍（具体事件，分配章节号）
   ↓
5. 汇总确认 → 生成 YAML
```

## 状态流转

**项目状态**：

```
planning → drafting → revising → completed
```

| 状态流转触发 | 条件 |
|-------------|------|
| planning → drafting | 大纲完善度 ≥ 85% 且章节规划完成 |
| drafting → revising | 正文写作进行中 |
| revising → completed | 用户确认完成 |
