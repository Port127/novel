# 任务

对单章进行结构化审查并给出可执行修订建议。

## 前置检查

1. 读取 `.current.yaml` 获取 `current_path`
2. 按 [章节自动推断协议](_protocols/chapter-auto-inference.md) 确定目标章节
3. 检查 `{current_path}/chapters/$0.md` 是否存在
4. 读取 `{current_path}/chapters/index.yaml` 获取章节元数据

## 输入参数

- `$0` (chapter_id): 目标章节 ID
- `--context`: 启用跨章衔接检查——读取前一章和后一章的摘要（`chapters/index.yaml` 中的 `summary`），检查本章与前后章的过渡是否连贯

## 执行步骤

### 1. 读取章节文本

提取本章大纲、正文、伏笔记录。

若指定了 `--context`：
- 读取 `chapters/index.yaml`，定位当前章的前一章和后一章
- 读取前后章的 `summary` 和 `characters_involved`（若已存在）
- 若前后章尚无 summary，读取其正文的首尾各 500 字作为替代

### 2. 审查维度

**审查要有文学品味，不只是打勾：**
- 指出"这里缺冲突"是合格的审查；指出"这里角色做了正确的选择但读者不会紧张，因为没有任何选项让他真正痛"才是好审查
- 表扬要具体——不说"开场不错"，说"开场用了一个反常细节制造悬念，这个细节同时暗示了角色的身份"
- 节奏问题用身体感受描述——"这三段读完像在水里走路"比"节奏偏慢"有用十倍

- 结构完整性：开场是否由异常、悬念或未解释信息起手；冲突是否升级；结尾是否留有钩子
- 节奏：信息密度与转折分布是否合理，过程性描写是否过多
- 情绪与张力：危险是否只和主角一起被发现，还是读者能提前感知；悲剧/情感戏是否建立在错过、误判、等待落空或反差脆弱上
- 角色一致性：是否符合角色卡设定，是否写出缺陷、执念或反差，而不是只有功能性对白
- 时间线一致性：与 `timeline/main.yaml` 是否冲突
- 伏笔：本章埋设与回收是否平衡
- **信息密度**：
  - 扫描每段，标记「纯设定/世界观介绍」段落（无角色动作、无冲突推进、无情绪变化的说明性文字）
  - 计算设定介绍段落占全章比例，超过 20% 即警告「信息灌输风险」
  - 检测单段内新名词/新概念数量，超过 3 个即标记「读者消化压力」
  - 给出融入建议：将设定信息拆散到角色行动、对白和冲突中，而非集中灌输
- **动作场景质量**（仅当章节包含战斗/追逐/紧张对峙时触发）：
  - 动作链多样性：是否只有「攻击→闪避→反击」单一循环
  - 环境互动：角色是否利用地形/道具/天气，还是在真空中打
  - 能力逻辑：招式/能力是否符合角色卡和力量体系设定
  - 感官层次：是否只有视觉，有没有声音/触感/疼痛/气味
  - 节奏变化：是否有快慢交替——爆发与喘息的切换
- **跨章衔接**（仅 `--context` 模式）：
  - 开场承接：本章开头是否自然接续上一章结尾的情绪/场景/悬念？是否让读者"断片"？
  - 信息重复：是否重复交代上一章已交代的信息？
  - 角色状态连续：上一章结尾角色的情绪/位置/伤势等状态是否在本章开头得到延续？
  - 节奏衔接：上一章结尾是高潮还是低谷？本章开头的节奏是否匹配（高潮后不宜再长段铺垫，低谷后可以缓起）？
  - 时间跳跃：若存在时间跳跃，是否给了读者足够的过渡信号？

### 3. 输出修订建议

给出 3-5 条"可直接改文"的建议，优先高收益项。

建议优先覆盖以下类型的问题：
- 删减拖慢节奏的过程描写
- 补强章首异常点或章尾钩子
- 补足代价、误判或信息差
- 让关键角色在情绪场里显出反差、软肋或执念
- 将集中灌输的设定信息拆散到行动和对白中
- 为模板化的动作场景补充环境互动和感官层次

## 输出格式

```
🔎 章节审查：$0

总评：{{评级}}

✅ 亮点
- ...

⚠️ 主要问题
- ...

🔧 修订建议（按优先级）
1. ...
2. ...
3. ...
```

## 注意事项

- 建议要具体到段落或情节点
- 不直接重写整章，优先提出可操作修改点
- 不把"多写惨"和"多写解释"当成强化情绪，优先强化钩子、代价与兑现
- `--context` 模式依赖前后章的 summary 字段；若前后章尚未推进到 draft/final，衔接检查基于首尾原文片段，精度会降低
- 建议写完一章后立即用 `--context` 检查与前一章的过渡，而非等全部写完再回头查
