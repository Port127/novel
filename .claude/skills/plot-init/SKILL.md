# 任务

初始化剧情大纲结构，创建 `outline.md`、`outline.yaml` 和 `plot_index.yaml`。

## 前置检查

1. 读取 `.current.yaml` 获取 `current_path`

## 输入参数

- `$0` (structure): 可选值：
  - **不传 / `auto`**（默认）：从 `ingestion_brief.md` 推导最适合的结构
  - `三幕式` / `英雄之旅` / `五段式` / `自定义`：手动快捷方式

> **推荐使用 `auto`**。手动指定模板适合"我很清楚自己要什么结构"的情况。
> 如果你有草稿或素材，更推荐走 `/pipeline-outline-bootstrap`，它包含素材消化 → 设定落地 → 结构推导的完整链路。

## 执行步骤

### 1. 检查现有大纲

读取 `{current_path}/plot/outline.md`。

如果已有内容，询问是否覆盖。

### 2. 确定结构

#### auto 模式（默认 / 推荐）

1. 读取 `{current_path}/ingestion_brief.md`
2. 从故事的独特机制和冲突升级路径推导结构——像导演分镜，不像学生填表：
   - 感受这个故事的呼吸——越来越快的窒息感，还是潮汐般的起落？
   - 这个故事的独特机制暗示了什么结构？
   - 核心冲突有几个自然升级台阶？
   - 主角认知有几个关键转折？
   - 想象读者在哪个点翻页最快、在哪个点放下书深呼吸——那就是结构的支点
   - 结构节点的名字从故事本身的意象或事件来，不用"第一幕""中点"这种教科书标签
3. 向用户呈现推导理由和推荐结构（可提供 1-2 个方案），等待确认
4. 若 `ingestion_brief.md` 不存在，提示用户先运行 `/draft-ingest` 或手动指定结构类型

#### 手动模式（指定具体结构名）

直接使用对应结构模板生成框架。适合用户已有明确结构偏好，不需要推导。

### 3. 生成叙述性大纲框架

写入 `{current_path}/plot/outline.md`。

auto 模式产出的框架基于推导结果（节点名称和分段来自故事本身，不是模板的固定标签）。
手动模式产出的框架基于模板（见下方"结构模板"节）。

### 4. 初始化结构化大纲

写入 `{current_path}/plot/outline.yaml`（若不存在）：

```yaml
premise: ""
theme: []
tone: []
structure: []
timelines: []
foreshadowing: []
pacing_curve: []
```

### 5. 初始化情节索引

写入 `{current_path}/plot/plot_index.yaml`（若不存在）：

```yaml
entries: []
```

### 6. 更新项目状态

更新 `state.yaml` 的 `plot.structure`。

## 结构模板（仅手动模式使用）

**三幕式：**
```markdown
# 大纲

## 第一幕：起（约25%）

### 开场
<!-- 日常世界，主角出场 -->

### 引发事件
<!-- 打破平衡的事件 -->

### 第一幕高潮
<!-- 主角踏上旅程 -->

---

## 第二幕：承转（约50%）

### 试炼
<!-- 挑战与成长 -->

### 中点
<!-- 故事中点的重大转折 -->

### 危机
<!-- 最低谷，最大挑战 -->

---

## 第三幕：合（约25%）

### 高潮
<!-- 最终对决 -->

### 结局
<!-- 新的平衡 -->
```

**英雄之旅：**
```markdown
# 大纲

## 第一阶段：启程
1. 平凡世界
2. 冒险召唤
3. 拒绝召唤
4. 遇见导师
5. 跨越第一道门槛

## 第二阶段：启蒙
6. 试炼、盟友、敌人
7. 接近最深的洞穴
8. 磨难
9. 奖赏
10. 返回之路

## 第三阶段：归来
11. 复活
12. 携带万灵药归来
```

## 输出格式

```
✅ 剧情结构初始化完成

📐 结构：{{结构名称或推导结果}}
📄 文件：plot/outline.md

已生成框架，接下来：
   /plot-add [章节] [内容]  添加情节
   /plot-suggest [描述]     获取AI建议
```

## 注意事项

- 结构可随时调整
- 支持混合结构
- auto 模式推导出的结构节点名称来自故事本身，不强制使用模板标签
- 如需更完整的从素材到大纲流程，使用 `/pipeline-outline-bootstrap`
